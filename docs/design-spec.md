# にほんご フラッシュカード — 設計仕様書

> 일본에 살게 되는 만 7세 한국 아이를 위한 일본어 플래시카드 웹앱

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **대상** | 만 7세 한국 남자 아이 (일본 거주 예정, 과학·지리 흥미) |
| **교사** | 줌 수업 주 3회 × 20분 |
| **보조** | 아이 엄마 (자율학습 보조) |
| **형태** | 정적 웹앱 (Vanilla HTML/CSS/JS) |
| **배포** | Vercel (무료 정적 사이트) |
| **일러스트** | 카와이 애니메이션풍 (しまじろう 스타일) |
| **저장소** | 독립 GitHub 레포 (기존 워크스페이스와 도메인 분리) |

## 2. 사용 시나리오

### 2-1. 줌 수업 모드 (선생님 주도)
1. 선생님이 줌에서 화면 공유
2. 레슨 선택 → 플래시카드 순서대로 진행
3. "이 글자가 뭘까~? 그림이 힌트야!" → 아이 답변 → 카드 뒤집기로 정답 확인
4. 카드 썸네일 바로 이전 레슨 카드 즉시 점프 가능 (복습 시)
5. 20분 동안 1행(5글자) + 복습 카드 2~3장

### 2-2. 자율학습 모드 (아이 + 엄마)
1. 아이가 직접 URL 접속 (태블릿/폰)
2. 홈에서 학습한 레슨 선택
3. 카드 앞면의 글자+그림을 보고 단어를 떠올린 뒤 뒤집어서 확인
4. 마지막 카드(5/5)에 도달하면 "다 봤어요! ✨" 버튼으로 진도 체크

## 3. 화면 구성

### 3-1. 홈 화면
- 앱 제목 + 카와이 마스코트 캐릭터
- Phase별로 그룹핑된 레슨 카드 그리드
- 각 레슨 카드 표시: 레슨 번호, 行 이름 (예: "あ행"), 완료 체크 ✓
- ~~50음도 차트 바로가기 버튼~~ → MVP 이후

### 3-2. 레슨 화면 (플래시카드)
- **카드 앞면**: 카와이 일러스트 + **학습 대상 히라가나/카타카나 1글자 크게 표시**
- **카드 뒷면**: 
  - ひらがな/カタカナ 전체 단어 (큰 글자)
  - 학습 중인 글자 하이라이트 (예: **あ**り 에서 あ를 강조)
    - `highlightIndex` 기반으로 위치 지정 (ん 등 비첫글자 대응)
  - 한국어 뜻
- **카드 썸네일 바**: 상단에 탭 가능한 작은 카드 인디케이터 (아무 카드나 즉시 점프)
- **네비게이션 버튼**: 
  - 이전/다음 버튼 (최소 56px, 아이콘+텍스트 병기) — **메인 네비게이션**
  - 스와이프: **보조 수단으로만** 지원
  - 키보드 좌/우 화살표 (데스크톱)
- **카드 뒤집기**: 카드 영역 탭으로 뒤집기
  - 첫 사용 시 "탭해서 뒤집어보세요!" 바운스 힌트 애니메이션 표시
- **"다 봤어요! ✨" 버튼**: 마지막 카드(5/5) 도달 시에만 표시 → 홈에 ✓ 반영
  - 뒤집기 여부는 강제하지 않음 (선생님/엄마가 오프라인 확인)
- 홈으로 돌아가기 버튼

### 3-3. 50음도 차트 (참고용) — MVP 이후
- 히라가나/가타카나 전체 표 (탭으로 전환)
- 학습 완료한 글자는 색상으로 구분
- 각 글자 클릭 시 해당 레슨 카드로 즉시 점프

## 4. 커리큘럼 설계

### 4-1. 교수법: 행(段) 순서 + 단어 페어링 하이브리드
- 일본 초등학교 표준 순서(あ→か→さ→...)를 따르되
- 각 글자마다 아이가 좋아할 만한 단어 + 카와이 일러스트를 페어링
- 한국어와 발음이 비슷한 단어를 전략적으로 배치 (친숙함 → 자신감)
- 아이의 과학·지리 흥미를 반영한 단어 선택 우선

### 4-2. 단계 구성 (총 25레슨, ~3개월)

| Phase | 내용 | 레슨 | 기간 |
|-------|------|------|------|
| 1 | 히라가나 기본 (あ~な행, 25자) | L1~L5 | 2주 |
| 2 | 히라가나 완성 (は~ん행, 21자) | L6~L10 | 2주 |
| 3 | 가타카나 기본 (ア~ナ행, 25자) | L11~L15 | 2주 |
| 4 | 가타카나 완성 (ハ~ン행, 21자) | L16~L20 | 2주 |
| 5 | **[TBD — Phase 4 완료 후 별도 설계]** | L21~L25 | 2주 |

> **Phase 5 확장 범위 (TBD)**:
> - 탁음(濁音): が行, ざ行, だ行, ば行 (20자)
> - 반탁음(半濁音): ぱ行 (5자)
> - 단어 복습 + 간단한 문장
> - 아이의 학습 진도와 상태를 보고 난이도 결정

### 4-3. 히라가나 ↔ 카타카나 시각적 분기
- 히라가나 Phase (1~2): 카드 배경색 **피치(peach)**
- 카타카나 Phase (3~4): 카드 배경색 **민트(mint)**
- 같은 단어 + 같은 일러스트 재활용, 글자만 카타카나로 전환

## 5. 기술 아키텍처

### 5-1. 프로젝트 구조
```
nihongo-flashcard/
├── index.html              ← SPA (싱글 페이지 앱)
├── style.css               ← 전체 디자인 시스템
├── app.js                  ← 앱 로직 + 커리큘럼 데이터 인라인
│                              - 커리큘럼 데이터 (JS 객체로 인라인 번들링)
│                              - hash 기반 라우팅 (#home, #lesson/1, #chart)
│                              - 카드 플립 로직
│                              - localStorage 진도 관리
│                              - 키보드/스와이프 네비게이션
├── images/
│   ├── mascot.png          ← 앱 마스코트 (스타일 레퍼런스)
│   ├── lesson-01/          ← あ행 이미지 (5장)
│   │   ├── ari.png

│   │   ├── enpitsu.png
│   │   └── onigiri.png

│   └── ...

```

> **변경**: `data/curriculum.json` 제거 → `app.js`에 커리큘럼 데이터 인라인.
> 이유: fetch 의존 제거 → 오프라인 내성 확보, 파일 수 감소.

### 5-2. 데이터 스키마 (app.js 인라인)
```javascript
const CURRICULUM = {
  appTitle: "にほんご フラッシュカード",
  phases: [
    {
      id: 1,
      title: "히라가나 기본",
      subtitle: "あ~な行",
      type: "hiragana",       // "hiragana" | "katakana" → 카드 배경색 분기
      lessons: [
        {
          id: 1,
          title: "あ행",
          row: "あ行",
          cards: [
            {
              character: "あ",
              word: "あり",
              wordReading: "あり",
              meaningKo: "개미",
              image: "images/lesson-01/ari.png",
              highlightIndex: 0    // 하이라이트할 글자 위치 (0-indexed)
            }
          ]
        }
      ]
    }
  ]
};
```

### 5-3. 핵심 인터랙션
1. **카드 플립**: CSS `transform: rotateY(180deg)` + `perspective` 3D 전환
2. **네비게이션**: 
   - **메인**: 이전/다음 버튼 (56px+, 아이콘+텍스트 병기)
   - **보조**: 모바일 터치 스와이프, 키보드 좌/우 화살표
   - **즉시 점프**: 카드 썸네일 바 (레슨 내)
   - ⚠️ 탭(뒤집기)과 스와이프(넘기기)의 제스처 충돌 방지: 수평 30px 이상 이동 시에만 스와이프 인식
3. **뒤집기 힌트**: 첫 사용 시 카드에 바운스 애니메이션 + "탭해서 뒤집어보세요!" 텍스트
4. **진도 추적**: `localStorage.setItem('completed', JSON.stringify([1,2,3]))` — 완료 레슨 ID 배열
   - 디바이스 로컬 저장만 지원 (크로스 디바이스 동기화 MVP 이후)
5. **레슨 완료**: 마지막 카드(5/5) 도달 시에만 "다 봤어요! ✨" 버튼 표시
6. **라우팅**: `window.location.hash` 기반 (#home, #lesson/1, #chart)
7. **하이라이트**: `highlightIndex` 기반으로 뒷면 단어의 특정 글자를 강조색으로 표시

## 6. 디자인 시스템

### 6-1. 색상 팔레트
```css
:root {
  --pink: #FFB5C2;       /* 메인 액센트 */
  --mint: #A8E6CF;        /* 보조 액센트 / 카타카나 카드 배경 */
  --lavender: #D4A5FF;    /* Phase 강조 */
  --peach: #FFDAB9;       /* 히라가나 카드 배경 */
  --sky: #87CEEB;         /* 링크/버튼 */
  --bg: #FFF8F0;          /* 전체 배경 */
  --text: #4A4A4A;        /* 본문 텍스트 */
  --highlight: #FF6B9D;   /* 학습 글자 하이라이트 */
}
```

### 6-2. 타이포그래피
- **일본어/한국어**: M PLUS Rounded 1c (Google Fonts) — 둥글고 친근한 서체
- **카드 앞면 글자**: 72~96px (한 글자를 크게 표시)
- **카드 뒷면 단어**: 48~64px (큰 사이즈로 가독성 확보)
- **한국어 뜻**: 24px

### 6-3. 카드 디자인
- 라운드 코너: 16px
- 소프트 섀도: `0 4px 20px rgba(0,0,0,0.1)`
- 뒤집기 애니메이션: 0.6s ease-in-out
- 카드 크기: 최소 320px × 400px (데스크톱), 반응형 축소
- **히라가나 카드**: 배경색 `--peach`
- **카타카나 카드**: 배경색 `--mint`

### 6-4. 버튼 디자인
- 최소 터치 타겟: **56px × 56px** (만 7세 아동 기준)
- 네비게이션 버튼: 아이콘 + 텍스트 병기 (예: ◀ 이전 / 다음 ▶)
- "다 봤어요! ✨" 버튼: 눈에 띄는 크기 + 축하 느낌의 색상

### 6-5. 배경
- 전체: 따뜻한 크림색 (#FFF8F0)
- 은은한 도트/별 패턴 (CSS 또는 SVG)

## 8. 이미지 제작 계획

### 8-1. 필요 이미지 수
- Phase 1 (L1~L5): 25장
- Phase 2 (L6~L10): ~21장
- Phase 3~4 (가타카나): Phase 1~2와 동일 단어 재활용 (같은 그림, 글자만 카타카나)
- Phase 5 (TBD): 미정
- 마스코트: 1장
- **총: ~50장 (고유 이미지) + 마스코트 1장**

### 8-2. 제작 방법 — 스타일 일관성 보장 전략
1. **마스코트 캐릭터를 먼저 1장 생성** → 전체 스타일의 레퍼런스 앵커로 사용
2. `generate_image` 도구로 카와이 스타일 일러스트 생성
3. **구체적 프롬프트 템플릿**으로 스타일 통일:
   ```
   "chibi kawaii [동물/사물], simple flat illustration, 
    thick black outline, pastel colors, white background, 
    round eyes, small blush marks on cheeks, 
    consistent with しまじろう art style, 1024x1024"
   ```
4. 마스코트 이미지를 **레퍼런스 이미지로 첨부**하여 세션 간 스타일 드리프트 방지
5. Phase 1의 25장을 **한 세션에서 연속 생성**
6. 배경 투명/흰색, 정사각형(1024×1024) 기본

### 8-3. 오프라인 내성
- `index.html`에 `<link rel="preload">` 태그로 모든 이미지 사전 로드
- 한 번 접속 후 브라우저 캐시에 이미지 보존
- 본격적인 PWA (Service Worker) 는 MVP 이후
